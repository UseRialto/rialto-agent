# Agent tools return Quote Comparison patch proposals

Rialto Agent will use one OpenAI Agents SDK-backed core with product-module tools rather than separate agents or a Python runtime in v1. Quote Comparison exposes mostly domain-level tools, with a small generic sheet-edit escape hatch, and those tools return patch fragments that the core aggregates into one Comparison Patch Proposal for the estimator to preview over the live sheet and approve or discard as a single unit. The same proposal can include visible workbook edits, formulas, highlights, and selection-state edits, but not external follow-up actions such as vendor notification, award, decline, or purchasing handoff. The product agent requires `OPENAI_API_KEY`; Rialto should not maintain a deterministic fallback planner as a parallel agent implementation. Agent responses must clearly distinguish completed, needs-clarification, blocked, and tool-error states, and an easily enabled Agent Debug Trace should show the turn's plan, tool calls, tool results, patch fragments, aggregated proposal, warnings, and errors in the webapp. This preserves visible user approval while letting the agent compose multiple backend sheet operations for one request.
