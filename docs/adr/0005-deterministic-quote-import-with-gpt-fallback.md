# Spreadsheet-first quote import with GPT-5.5 normalization for other files

External Quote Import uses one user-facing quote upload path. CSV, TSV, XLS, and XLSX files go directly through deterministic spreadsheet extraction and deterministic quote-row parsing. PDF, XML, text, HTML, YAML, and any other non-spreadsheet quote files get a fast deterministic preflight after text extraction. If that preflight can already build priced quote rows, Rialto imports immediately without waiting on GPT-5.5. If not, GPT-5.5 normalizes the file into importer-ready CSV text before it enters the deterministic quote importer.

The model normalizer must return only facts present in the source file and must self-check the normalized CSV element by element against the original source text. Its output still re-enters the normal importer, so quote comparison shape, line matching, per-thousand unit-price normalization, imported-note handling, and alternate/substitution flags remain centralized. Imported quote notes such as "alternate manufacturer" or "substitution" are preserved as notes only and do not set `is_alternate`.

For PDFs, Rialto may use non-LLM readers to produce source text for either the deterministic preflight or GPT-5.5 conversion. The current source-text path tries the repo PDF.js runtime first and then an independent `pdf-parse` text extractor if the runtime fails or returns no text. Either the extracted text or model-normalized CSV then feeds the same schema-validated quote row parser used by spreadsheets.

The agent-facing Quote Comparison tool surface must expose the same visible workbook operations the UI supports. Sorting is therefore available as an Excel-like sort proposal tool in addition to the broader sheet-structure edit tool, and formula-dependent edits such as Unit Price recomputing Total Price should flow through Comparison Sheet patch/override state rather than changing hidden bid source data.
